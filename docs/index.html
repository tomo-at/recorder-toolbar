Prototype Settings

Configuration options

V5 exposes three independent axes — Default style, Recording, and Uploading. Each can be changed at runtime from Prototype Settings without restarting, producing 5 × 3 × 4 = 60 distinct combinations to explore.

Default style

Step by step
Recording type buttons are shown first. Camera and microphone controls are hidden until a type is selected, minimising the number of controls visible at a glance.
Revealed all
All controls — recording type, camera, mic, and settings — are visible from the initial state. Reduces the number of steps for users who already know what they want.
Revealed all (compact)
Same as Revealed all but with a denser layout and tighter label spacing. Keeps all controls accessible while reducing the toolbar's total width.
Message
A 16 px contextual header sits above the main controls, showing a short message that updates per state — e.g., "Choose a recording type" or "Recording in progress". Toolbar height increases to 66 px.
Horizontal layout
All controls are arranged in a single horizontal row at a reduced 48 px height. Prioritises a low vertical footprint over label legibility.

Recording

Toolbar
Window and display selection is handled directly in the toolbar. After the user clicks a target, a Record button appears inline — the confirmation step stays within the toolbar.
Select to start
Clicking a window or display immediately starts the recording countdown, skipping the explicit Record button. Optimised for speed.
Selected region
A persistent indicator shows the selected region inside the toolbar. The user can review their selection before tapping Record to begin.

Uploading

Toolbar
A 2 px progress bar overlays the top edge of the toolbar while the file uploads. On completion, a badge appears on the Settings icon and the All Videos count increments.
Toolbar + Complete message
Same 2 px progress bar during upload. On completion, a small banner floats 8 px above the toolbar showing "Upload complete", a View video link, and a dismiss button.
Menu bar + Notification
Upload progress is shown as an animated arc around the menu bar icon. A system notification is delivered on completion, with the menu bar icon briefly showing a checkmark.
Upload mode
The entire toolbar is replaced by a dedicated upload UI showing a labelled progress bar. On completion, the toolbar shows "Upload complete ✓" with View video and New recording actions.

Overview

What this prototype validates

A floating NSPanel toolbar that sits above all windows on macOS. The goal is to validate the full recording-setup flow — type selection, window/display picking, camera & microphone configuration, settings, and countdown — before committing to a production implementation. No actual screen capture or recording takes place.

What changed in Variation 2

  • Camera and microphone segments are visible from the very first state — no need to select a recording type first to see device controls.
  • Toolbar width at typeSelect increases from 389 px to 550 px to accommodate the always-on device area.
  • After confirming a window or display target, the toolbar expands to 623 px and a Record button appears in-line — reducing the number of distinct toolbar states the user experiences.
  • Cam-only button icon changed from a live camera thumbnail to the person.crop.rectangle.fill SF Symbol, matching the Figma design.

What changed in Variation 3

  • The four recording-type buttons are grouped into a single Segmented Control — a pill-shaped container with rgba(0,0,0,0.24) background, 2 px internal padding, and 10 px corner radius.
  • The active segment is highlighted with rgba(1,1,1,0.16) and an 8 px inner corner radius (outer − padding).
  • Toolbar width at typeSelect grows by 4 px to 554 px to accommodate the container's left/right padding.
  • Camera, mic, settings, and keyboard shortcut tooltips carry over from Variation 2.

What changed in Variation 4

  • A 16 px message header bar sits above the controls, showing a contextual message that changes per state: Choose a recording type, Click a display/window to start recording, Click Record when you're ready, Starting in N…, Recording in progress.
  • The X (close) button is removed from all toolbar states — total height increases from 56 px to 66 px, while typeSelect width shrinks from 554 px to 482 px.
  • The Settings menu gains a Divider + Quit Airtime option at the bottom, which calls NSApp.terminate(nil).
  • The segmented control wrapper background is removed; type-selector items sit directly in the HStack with per-item hover highlights only.

What changed in Variation 5

  • All layout, recording-flow, and upload-feedback styles are unified into a single build controlled by three independent axes in Prototype Settings — no need to switch build targets to compare variations.
  • New Horizontal layout default style: a compact 48 px single-row toolbar with no header bar.
  • New Revealed all (compact) default style: a denser version of Revealed all with tighter icon and label spacing.
  • New Upload mode: the entire toolbar is replaced by an upload progress view, transitioning to a completion state with View video and New recording actions on finish.
  • New Toolbar + Complete message: a floating 24 px banner appears above the toolbar on upload completion, with a View video link and a dismiss button.

Interaction flow

States & transitions

The toolbar progresses through five states. Each state resizes the panel and updates the controls shown.

  1. 1 Type Select — Initial state. Choose Display, Window, Area, or Cam only. Hovering each button shows a preview overlay. Clicking Display or Window activates a selection overlay without changing the toolbar — the button highlights to show the active mode.
  2. 2 Window / Display overlay — A full-screen dim overlay appears. The hovered window or display is highlighted with an orange border. Click a target to confirm; the toolbar switches to the Record state. Press Esc or re-click the same button to cancel.
  3. 3 Record state — Camera and microphone segments appear with device pickers. A Settings panel opens above the Settings button and shows Camera, Mic, and Countdown sub-panels on hover. Press Record to begin.
  4. 4 Countdown — A full-screen number overlay counts down (3, 2, 1 or 1, depending on the Settings choice). The toolbar shrinks to 297 px. Press Esc to cancel.
  5. 5 Recording — Toolbar shows a live timer, Pause / Resume, Restart, and Stop. Stopping returns to Type Select and triggers the upload flow configured in Prototype Settings.
